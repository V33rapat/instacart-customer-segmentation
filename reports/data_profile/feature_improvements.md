# แนวทางการปรับปรุง Feature

- รวมข้อมูลคำสั่งซื้อเป็น Feature ระดับผู้ใช้ (ความถี่การสั่งซื้อ, ค่าเฉลี่ย/มัธยฐานขนาดตะกร้า)
- สร้าง Feature ความใหม่ (จำนวนวันตั้งแต่สั่งซื้อครั้งล่าสุด, เปอร์เซ็นไทล์หมายเลขคำสั่งซื้อ)
- วิเคราะห์รูปแบบเชิงเวลา (ค่า entropy ของชั่วโมง/วันในสัปดาห์ที่สั่งซื้อ, ช่วงเวลาที่ชอบซื้อของ)
- คำนวณตัวชี้วัดความหลากหลายของสินค้า (จำนวน aisle/department ที่ไม่ซ้ำกันต่อผู้ใช้)
- บันทึกพฤติกรรมการสั่งซ้ำ (อัตราส่วนการสั่งซ้ำ, อัตราการซื้อซ้ำตามหมวดหมู่)
- ใช้ days_since_prior_order เพื่อคำนวณความแปรปรวนของระยะเวลาระหว่างการสั่งซื้อในระดับผู้ใช้
- แบ่ง order_hour_of_day เป็นช่วงเวลาในวัน เพื่อลดความเบาบาง (sparsity)
- แปลง order_dow เป็น Feature แบบวงกลม (sin/cos)
